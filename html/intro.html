시스템을 단단하게 설계하고, 문제가 생겨도 빠르게 복구할 수 있는 능력을 갖춘 네트워크 엔지니어로 성장하고 싶습니다.

프로젝트 오류 해결

공주대학교 행정 앱

게시판 화면 구현 중 게시판 화면 UI불러올 때,
생명주기 onCreate에서 NULLPOINTER 오류 발생

👉원인👈

: 네트워크 요청 처리 전, 즉 서버에서 아직 결과가 도착하기 전에 데이터 처리를 시도하여 오류 발생함
데이터를 받아서 UI를 띄우는 로직을 수행하는 상황인데 데이터가 더 늦게 도착하는 상황 발생
이로 인해 UI 설정 시 아무런 데이터를 받지 못하고 UI 업로드를 해서 NULL POINTER 오류가 일어남
이는 서버에서 데이터를 받아 올때 retrofit의 enqueue()함수를 통해 데이터를 받아오기 떄문에 발생함
retrofit의 enqueue()는 비동기화로 이루어지기 때문에 서버 응답이 오기 전에 다음 처리를 시도하게 됨
이로 인해 데이터가 다 오기 전에 UI업로드를 진행하여 nullPointer오류가 발생한 것이었음

👉해결방식👈

비동기와 동기의 혼합으로 해결


동기

: retrofit의 execute()를 이용하여 네트워크 요청을 동기화하여 응답을 받은 후 데이터 처리를 하도록 수정함


비동기

: 서버 응답 시까지 UI를 띄우지않으면 사용자 입장에서는 답답할 수 있기 때문에 UI를 띄우는 메인 스레드와
네트워크 요청을 처리하는 백그라운 스레드로 나누어 비동기화로 수행되도록 함

👉아쉬운 점👈

: 새로고침 기능과 데이터 응답 대기 아이콘을 추가하면 사용자 편의성이 극대화되었을 것
이는 서버 배포까지 하지못해서 구현하지 못함

관심 기술

👉스프링/스프링 부트👈

스프링은 웹 애플리케이션을 쉽고 빠르게 만들 수 있도록 도와주는 자바의 웹 프레임워크이다.
스프링 부트는 스프링(Spring) 프레임워크에 톰캣(Tomcat)이라는 서버를 내장하고 여러 편의 기능들을 추가하여 개발자들 사이에서 꾸준히 인기를 누리고 있다.
톰캣은 클라이언트의 요청을 해석하여 그에 맞는 자바 프로그램을 실행한 후 그 결과를 응답해 주는 웹 애플리케이션 서버이다.
또한, 스프링/스프링 부트는 기본 보안이 잘되어있어있는 튼튼한 웹 프레임워크라고 할 수 있다.
이러한 장점으로 인해 전자정부표준프레임워크로 지정되었으며, 많은 기업에서 사용하고 있다.
자바 프레임워크인 만큼 자바에 대한 높은 이해도가 필요하며 최근에는 코틀린을 지원하여 코틀린+스프링 조합으로 개발하는 사람들도 늘고 있다고 한다.

👉핀테크👈

핀테크(fintech)는 Finance(금융)와 Technology(기술)의 합성어다.
말 그대로 금융 서비스와 정보기술(IT)의 융합을 통한 금융서비스 및 산업의 변화를 통칭한다
특히 모바일을 통한 결제, 송금, 자산관리, 크라우드 펀딩 등 금융과 IT기술의 융합을 의미한다
모바일뱅킴, 앱카드, 애플페이, 삼성페이 등을 예시로 들 수 있다
예민한 돈을 다루는 기술인 만큼 높은 보안성과 안전성이 요구된다
또한 개발자들은 단순 개발 지식뿐만 아니라 금융 지식이 요구되는 경우도 많다
주요 기술로는 인공지능, 블록체인, 클라우드, 빅데이터 등이 있다
