성장 과정

컴퓨터 관련 전공에 대한 관심은 경영학과 입학 전부터 꾸준히 가지고 있었습니다.
코딩 동아리 PRIMITIVE 활동을 통해 실제로 프로그래밍을 접하면서 소프트웨어학과로 전과하게 되었고,
Java, Python, C 등 다양한 언어를 배우는 과정에서 데이터 처리 흐름과 백엔드 시스템의 구조에 흥미를 느끼게 되었습니다.
특히 2022년 발생한 카카오의 개인정보 유출 사건은 저에게 큰 충격이었습니다.
수많은 이용자의 서비스가 마비되고 개인정보가 유출되면서, IT 인프라의 취약점이 사용자 삶에 얼마나 큰 영향을 미칠 수 있는지를 실감했습니다.
이 사건을 계기로 단순히 프로그램을 만드는 것에서 그치지 않고,
서비스가 운영되는 인프라, 즉 클라우드 시스템의 중요성과 책임에 대해 깊이 고민하게 되었습니다.
이후 네트워크 구조, 보안, 장애 대응 시스템에 대해 더 공부하게 되었고,
안정적인 인프라를 설계하고 운영할 수 있는 클라우드 개발자가 되고 싶고, 큰 환경에서 신뢰성을 책임지는 개발자가 되고자 결심하게 되었습니다.
앞으로는 백엔드 개발자로서의 역량뿐 아니라, 보안과 인프라에 대한 지식까지 갖춘 실력 있는 클라우드 개발자로 성장해나가고 싶습니다.

시스템을 단단하게 설계하고, 문제가 생겨도 빠르게 복구할 수 있는 능력을 갖춘 네트워크 엔지니어로 성장하고 싶습니다.

프로젝트 오류 해결

planJ

게시판 화면 구현 중 게시판 화면 UI불러올 때,
생명주기 onCreate에서 NULLPOINTER 오류 발생

👉원인👈

: 네트워크 요청 처리 전, 즉 서버에서 아직 결과가 도착하기 전에 데이터 처리를 시도하여 오류 발생함
데이터를 받아서 UI를 띄우는 로직을 수행하는 상황인데 데이터가 더 늦게 도착하는 상황 발생
이로 인해 UI 설정 시 아무런 데이터를 받지 못하고 UI 업로드를 해서 NULL POINTER 오류가 일어남
이는 서버에서 데이터를 받아 올때 retrofit의 enqueue()함수를 통해 데이터를 받아오기 떄문에 발생함
retrofit의 enqueue()는 비동기화로 이루어지기 때문에 서버 응답이 오기 전에 다음 처리를 시도하게 됨
이로 인해 데이터가 다 오기 전에 UI업로드를 진행하여 nullPointer오류가 발생한 것이었음

👉해결방식👈

비동기와 동기의 혼합으로 해결


동기

: retrofit의 execute()를 이용하여 네트워크 요청을 동기화하여 응답을 받은 후 데이터 처리를 하도록 수정함


비동기

: 서버 응답 시까지 UI를 띄우지않으면 사용자 입장에서는 답답할 수 있기 때문에 UI를 띄우는 메인 스레드와
네트워크 요청을 처리하는 백그라운 스레드로 나누어 비동기화로 수행되도록 함

👉아쉬운 점👈

: 새로고침 기능과 데이터 응답 대기 아이콘을 추가하면 사용자 편의성이 극대화되었을 것
이는 서버 배포까지 하지못해서 구현하지 못함

관심 기술

👉스프링/스프링 부트👈

스프링(Spring)은 자바 기반의 대규모 웹 애플리케이션을 쉽게 만들 수 있도록 지원하는 프레임워크다.
스프링 부트(Spring Boot)는 스프링을 더 간편하게 사용할 수 있도록 설정을 자동화해 빠르게 개발할 수 있게 해준다.
네이버 클라우드는 스프링/스프링 부트로 만든 웹 서비스를 클라우드 서버에 배포하고 운영할 수 있는 인프라를 제공한다.
예를 들어, 네이버 클라우드의 Compute, Cloud DB, Object Storage 등을 활용해 백엔드 서비스와 데이터베이스를 연동할 수 있다.
스프링 부트 앱을 Docker 이미지로 만들어 네이버 클라우드 Kubernetes 서비스에 배포하면 효율적인 클라우드 운영이 가능하다.

👉mysql👈

MySQL은 오픈소스 관계형 데이터베이스로, 다양한 웹 애플리케이션에서 널리 사용된다.
네이버 클라우드는 ‘Cloud DB for MySQL’ 서비스를 통해 MySQL을 클라우드 환경에서 쉽게 사용할 수 있도록 지원한다.
이 서비스는 자동 백업, 보안 설정, 모니터링 등 관리 기능을 포함해 안정적인 데이터 운영을 가능하게 한다.
따로 MySQL을 설치하지 않아도 콘솔에서 몇 번의 클릭으로 데이터베이스를 구축하고 바로 연동할 수 있다.
스프링 부트와도 잘 연동되어, 네이버 클라우드를 활용한 백엔드 개발에 매우 적합한 DB 솔루션이다.
